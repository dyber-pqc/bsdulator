The “one killer demo”

Stateful, security-critical network services as isolated units on a Linux host with ZFS snapshots for atomic rollback.

  • Each jail-like unit = its own filesystem dataset
  • Snapshot before change → apply → rollback if needed
  • Deterministic networking (appliance-grade)

Architecture

Linux host
├─ bsdulator runtime (CLI + supervisor)
├─ namespace/cgroup isolation
├─ ZFS datasets per unit
└─ “jail” lifecycle (create/start/snapshot/rollback)

Keep this narrowly scoped. Don’t pitch “Docker replacement”.

Demo: snapshot → break → rollback

This is the story you want in front of platform/network engineers. Keep it short and copy/pasteable.

1) Install

curl -fsSL https://example.com/install.sh | sudo sh

2) Create a unit (example: DNS)

sudo bsdulator create dns01 --dataset tank/bsdulator/dns01 --net vnet0

3) Snapshot before change

sudo bsdulator snapshot dns01 pre-change

4) Apply a change (intentionally break it)

sudo bsdulator exec dns01 -- sh -lc 'echo "broken" > /etc/resolv.conf'

5) Rollback atomically

sudo bsdulator rollback dns01 pre-change

Replace these commands with your real CLI once implemented. The demo page should match reality.
